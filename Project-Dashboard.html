Measuring Data Exposure in LLM-Integrated Productivity Tools

How much of a confidential document is silently transmitted to a writing-assistant's servers when its browser extension is running by default — even after the user is told not to share the file with AI tools.

And it's not a one-tool fluke: LanguageTool, an independent grammar checker, silently transmitted 91.9% and all 12 secrets too — replicating the finding across two separate tools.

Where the data went

Both grammar checkers shipped the document to their own servers.

Destinations: capi.grammarly.com (WebSocket /freews) and api.languagetool.org (POST /v2/check). 100% of intercepted events used HTTPS; 0 TLS-handshake failures — nothing hidden by certificate pinning.

How the measurement works

Threat model: a user pastes a received confidential file into a normal text field
1

Plant secrets

A synthetic 2,065-char memo carries 12 unique fictional identifiers, incl. a UUID canary.

2

Paste it

Into a plain field on a local page, with the tool's extension running by default.

3

Intercept

mitmproxy decrypts the HTTPS traffic leaving Firefox on Kali Linux.

4

Measure

A capture add-on scans traffic for the document (20-char windows) & the 12 secrets.

5

Compare

Tool run vs a no-extension baseline, repeated 5× for reproducibility.

Headline metric: how many of the 12 planted secrets reached the servers (the canary makes any match unambiguous). Supporting metric: % of document characters transmitted. Exposure counts outbound traffic only (client → server).

Tool progress

Two independent automatic grammar checkers, plus a no-extension control
Scope. The study measures automatic / background transmission — tools that send the document with no user action beyond pasting. Two independent grammar checkers (Grammarly, LanguageTool) are compared against a no-extension control. On-demand paraphrasers were considered but fall outside this scope: they only transmit text the user actively submits, so they don't pose the same passive-exposure risk.
